# Phase 1 2016 Full Archive

This archive is the cleaned reproducible Phase 1 package requested for the
2016-start research record.

It keeps only:

1. factor attribution (`FF5 + MOM`);
2. `phase1_smooth_score_policy_v1` and all of the steps reported inside that
   report.

It excludes older exploratory branches such as state sorting, predictive
regression, old OOS validation, old state-action policy, and all ElasticNet
experiments.

## Data Start

- G/D source return sample starts on `2016-12-21`.
- Factor attribution regression sample starts on `2016-12-21`.
- Smooth score policy evaluation starts from the G/D source sample start,
  `2016-12-21`.
- Smooth score features that require `G-D trailing 126d` naturally become
  available after the 126-trading-day warmup. This warmup is retained and
  documented, not treated as a mid-sample cut.

## Reproduce

Run from the repository root:

```bash
python scripts/run_phase1_factor_attribution.py
python scripts/run_phase1_state_framework_v2.py
python scripts/run_phase1_smooth_score_policy_v1.py
python scripts/build_phase1_2016_full_archive_report.py
```

Or:

```bash
./reproduce.sh
```

## Main Combined Report

The single merged report is:

```text
data/phase1/archive_2016_full/reports/phase1_2016_full_combined_report.md
```

It combines the full factor attribution report and the full smooth score policy
report, with an opening section documenting data lineage and exclusions.

## Large Regenerable Intermediates

The following files are intentionally excluded from the archive package because
they are very large and can be regenerated by `run_phase1_smooth_score_policy_v1.py`:

- `smooth_score_policy_v1_signals.csv`
- `smooth_score_policy_v1_daily_returns.csv`
- `smooth_score_policy_v1_supplementary_tilt_common_oos_returns.csv`

All summary tables, reports, plots, validation outputs, and lineage tables are
included.
